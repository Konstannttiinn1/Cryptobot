# Cryptobot

Telegram-бот на Python для личного мониторинга курсов криптовалют. Бот позволяет выбрать монеты, валюту отображения RUB или USD, интервал уведомлений и получать актуальные курсы с изменением за 24 часа.

## Возможности

- `/start` и `/menu` — приветствие, главное меню и компактная нижняя клавиатура быстрого доступа.
- `/price` — разовый запрос актуальных курсов выбранных монет.
- `/coins` — выбор монет из каталога с пагинацией и поиском.
- `/settings` — текущие настройки пользователя.
- `/stop` — отключить уведомления.
- `/hide` — скрыть нижнюю клавиатуру.
- `📊 Курс сейчас` — разовый запрос актуальных курсов выбранных монет.
- `🪙 Выбрать монеты` — выбор монет галками из каталога 100 монет, с пагинацией по 10 монет и поиском. Максимум — 20 монет на пользователя.
- `💱 Валюта` — выбор RUB или USD.
- `⏱ Уведомления` — выбор интервала 5, 10, 15, 30 или 60 минут автоматически включает рассылку; пункт `🚫 Не присылать уведомления` отключает её.
- `🆘 Поддержка` — ссылка из `SUPPORT_URL`.

Бот только показывает курсы и отправляет уведомления. Торговля, биржевые аккаунты и биржевые API-ключи не используются.

## Стек

- Python 3.11+
- aiogram 3.x
- SQLite + aiosqlite
- aiohttp
- python-dotenv
- async scheduler внутри приложения
- CoinGecko Demo API

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Настройка `.env`

Скопируйте пример и заполните значения:

```bash
cp .env.example .env
```

```env
BOT_TOKEN=123456:telegram_bot_token
COINGECKO_API_KEY=your_demo_key
SUPPORT_URL=https://t.me/username
MAX_SELECTED_COINS=20
```

- `BOT_TOKEN` — токен Telegram-бота от [BotFather](https://t.me/BotFather).
- `COINGECKO_API_KEY` — необязательный Demo API key CoinGecko. Если ключ указан, бот отправляет его в заголовке `x-cg-demo-api-key`.
- `SUPPORT_URL` — ссылка на поддержку. Если не задана, бот покажет сообщение, что ссылка не настроена.
- `MAX_SELECTED_COINS` — максимальное количество монет, которое может выбрать один пользователь. По умолчанию — 20.

### Как получить CoinGecko API key

1. Откройте сайт CoinGecko API.
2. Зарегистрируйтесь или войдите в аккаунт.
3. Создайте Demo API key в личном кабинете.
4. Добавьте ключ в `.env` как `COINGECKO_API_KEY`.

## Запуск

```bash
python -m app.main
```

При первом запуске будет создан файл SQLite-базы `bot.db` с таблицами:

- `users` — настройки пользователей.
- `user_coins` — выбранные пользователем монеты.
- `price_cache` — последний полученный кэш цен.

Можно переопределить путь к базе переменной окружения `DATABASE_PATH`.

## Выбор монет

В каталоге доступно 100 популярных монет CoinGecko. Раздел `🪙 Выбрать монеты` показывает их постранично: по 10 монет на странице, с кнопками перехода назад/вперёд и кнопкой `🔎 Поиск`. Поиск работает по тикеру, названию и `coin_id`; найденные монеты можно включать и выключать так же, как в основном списке.

Один пользователь может выбрать максимум 20 монет для отслеживания. Ограничение нужно, чтобы уведомления оставались читаемыми и не превышали лимит обычного Telegram-сообщения. Если переменная `MAX_SELECTED_COINS` не задана в окружении, используется значение `20`.

## Пример сообщения

```text
📊 Курс сейчас

₿ BTC: 9 420 000 ₽ | +1.8% за 24ч
🔵 GRAM: 305.20 ₽ | -0.6% за 24ч
💵 USDT: 89.40 ₽ | +0.01% за 24ч

Валюта: RUB
Обновлено: 14:35
```

## Источник данных

Используется CoinGecko endpoint:

```text
GET https://api.coingecko.com/api/v3/simple/price
```

Параметры:

- `ids` — список coin ids через запятую.
- `vs_currencies=rub,usd`.
- `include_24hr_change=true`.

Бот не делает отдельный запрос на каждую монету: выбранные `coin_id` собираются в один пакетный запрос.

## Устойчивость

- Таймаут API-запроса — 10 секунд.
- Ошибки CoinGecko логируются.
- Если API недоступен, пользователь получает сообщение: `Не удалось получить курс. Попробуйте позже.`
- Старые inline-кнопки обрабатываются без падения приложения.
- Уведомления не отправляются, если у пользователя нет выбранных монет.
- Планировщик собирает выбранные монеты пользователей в один пакетный запрос к CoinGecko за проход рассылки, а не делает отдельный API-запрос для каждого пользователя.
